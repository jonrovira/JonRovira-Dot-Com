Composite

Technology as urban art

  • Front-End Development
  • UI Design

Harnessing social media feeds, computer vision technology, and the Google Maps API, Composite provides an artistic representation of demographic and sociological trends in different regions.

  • GitHub Repository

Background

Alex Kaldjian, Matt Ehringer, and I began working on Composite in the spring of 2014 when our professor, Zach Wise, proposed the idea of an interface displaying "location-based-faces" in our class titled Innovations in Journalism and Technology. We ultimately dreamed of putting multiple instances of the inteface on various street corners with their configurations set to those locations. The first step, though, was building out the application for a desktop.

The Technology

By far, the most unique part of this project is the software architecture that's working in the background. This is primarily due to the complexity of the technology stack, which features a Flask backend taht's constantly interacting with the Instagram API and OpenCV, and an Angular frontend that heavily utilizes the Google Maps API's capabilities. The flow below describes everything that happens throughout one user story.

  • The user drags the map to a new location.
  • Google Maps calculates the new center of the map as well as the radius from that center shown in the map.
  • Python uses that location information as well as the date in the input field in the interface to query the Instagram API for the appropriate images.
  • OpenCV uses facial-recognition to crop out every single face from each of those raw images.
  • Flask passes the resulting images of individual faces back to Angular to display to the user.

End Result

By the end of the quarter, our working prototype of Composite was well on its way, with each of the features (image cropping, maps functionality, etc.) working individually but only to a limited capacity when put together. I intend to continue working on the project myself towards the goal that we originally had in mind.
